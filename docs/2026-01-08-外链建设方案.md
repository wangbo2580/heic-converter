## Key Technical Decisions

### Why Server-Side Processing?

Most browser-based converters use client-side JavaScript libraries. I chose server-side processing with Sharp because:

1. **Better format support** - Sharp handles HEIC/AVIF natively via libvips
2. **Consistent results** - No browser compatibility issues
3. **EXIF preservation** - Sharp's `withMetadata()` makes this trivial

```typescript
// Simplified conversion logic
const result = await sharp(inputBuffer)
  .toFormat(outputFormat, { quality })
  .withMetadata() // Preserves EXIF
  .toBuffer();
```

### Handling Batch Processing

The API accepts FormData with multiple files and processes them in parallel:

```typescript
const results = await Promise.all(
  files.map(async (file) => {
    // Process each file
  })
);
```

### Security Considerations

- File signature validation (magic numbers)
- Size limits (50MB per file, 100 files max)
- Files deleted immediately after response
- CORS restricted to own domain
